Rails 4 Directory Structure
Jun 30th, 2013

Rails 4 is released. Lots of new features has been added to rails. Here we will see what and why new directories or files rails 4 added.

We will create a sample app in rails 4 so that it will be easy to see the new structure. I am assuming you have updated your rails. To check current version of rails run following command:

It should be Rails 4.0.0

Now create a very simple rails application

It creates bunch of files and directories. You will see there are some new directories and files. Lets go to each one by one

app/

In this directory you will see new concerns/ dir under controllers/ and models/. Concerns are added to manage your models/controllers in a better way . You can

Make your models ligh-weighted. Cross sharing of modules code.

Mixins and modules code can be put under this dir. Lets say in our blog application we have user model and post model and we require to add comments for both model so we created a commentable module in our concerns/ dir and included in both model.

Now suppose we need to add one more module notable to our model post.

Now you can include Commentable to Noteable

Now both Commentabel and Notebale are accessible for Post class.

  • bin/
    • App executable bundle, rails and rake are under this dir. So dont need to run ‘bundle exec’ before any rails command.   • doc/
    • Doc has been removed from rails directory. Its is extracted in “sdoc” gem and is added to Gemfile by default so still you can generate docs by running following rake task

  • script/
    • Its removed and rails executable is moved to bin/ dir

  • test/
    • This directory has been changed differently.

    • Now we have controller and models in place of functionals and units
    • We have helpers and mailers dir under test
    • Performance test has been extracted to a new Gem. So performance dir is not under test by default
